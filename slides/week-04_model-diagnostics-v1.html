Key Concepts and Definitions

What are Residuals?

  • Definition:
    The difference between the observed value \(Y_i\) and the fitted (predicted) value \(\hat{Y}_i\):

\[ e_i = Y_i - \hat{Y}_i \]

  • Importance:
    Residuals capture the unexplained variation after fitting the model.

Example: Calculating a Residual

# Example: Fitting a simple linear model and calculating residuals
set.seed(123)
x <- rnorm(20, mean = 5, sd = 2)
y <- 2 + 3 * x + rnorm(20)
model_example <- lm(y ~ x)
residuals_example <- resid(model_example)
head(residuals_example)
         1          2          3          4          5          6 
-1.0715154 -0.1958163 -0.8638412 -0.6831998 -0.5747476 -1.5122930
  • Discussion:
    Check the output to see the first few residuals calculated from the model.
Note

resid() extracts residuals from a linear model object.

Quiz: What is a Residual?

  • (A): The difference between the observed and fitted values.
  • (B): The sum of the observed and fitted values.
  • (C): Only the observed value.
  • (D): None of the above.

Standardized Residuals

  • Definition:
    Standardized residuals scale raw residuals by the estimate of the standard deviation:

    \[ z_i = \frac{e_i}{\sqrt{\text{MSE}}} \]

    where

    \[ \text{MSE} = \frac{\sum_{i=1}^{n} e_i^2}{n-p} \]

  • Usage:
    Identifies outliers; values of \(|z_i| > 3\) are typically flagged.

Example: Standardized Residuals Calculation

# Calculate standardized residuals from the previous model
mse <- sum(residuals_example^2) / (length(x) - length(coef(model_example)))
std_residuals <- residuals_example / sqrt(mse)
head(std_residuals) # Display the first few standardized residuals
         1          2          3          4          5          6 
-1.2619625 -0.2306199 -1.0173771 -0.8046292 -0.6769011 -1.7810823 
head(rstandard(model_example)) # Using the rstandard() function
         1          2          3          4          5          6 
-1.3138482 -0.2375746 -1.1111842 -0.8256544 -0.6944889 -1.9762101

    Compare the output from the manual calculation and the rstandard() function.

Quiz: Standardized Residuals Interpretation

  • (A): They indicate the scaled difference between observed and fitted values.
  • (B): They are the same as the raw residuals.
  • (C): They measure model fit directly.
  • (D): They are irrelevant for outlier detection.

Homoscedasticity: Constant Variance

  • Definition:
    The variance of residuals remains constant across levels of the predictor(s).
  • Visual Check:
    A residual vs. fitted plot should display a random cloud of points.
  • Implication:
    Heteroscedasticity can lead to biased standard errors.

Visualizing Homoscedasticity

# Plot residuals vs. fitted values for the example model
plot(model_example$fitted.values, resid(model_example),
    xlab = "Fitted Values", ylab = "Residuals",
    main = "Residuals vs. Fitted Values"
)
abline(h = 0, col = "red")
  • Observation:
    Look for any funnel or pattern indicating non-constant variance.

Quiz: Homoscedasticity Concept

  • (A): Residual variance is constant.
  • (B): Residual variance increases with fitted values.
  • (C): Residuals are normally distributed.
  • (D): Residuals are independent.

Normality of Errors

Definition:

Errors (residuals) should be approximately normally distributed.

  • Diagnostic Tool: Q-Q plots compare the distribution of residuals to a normal distribution.
  • Mathematical Note: In a Q-Q plot, if residuals lie on a 45° line, normality is supported.

Visualizing Normality: Q-Q Plot

qqnorm(resid(model_example), main = "QQ-Plot of Residuals")
qqline(resid(model_example), col = "blue")
  • Interpretation: Deviations from the line indicate departures from normality.

Quiz: Q-Q Plot Interpretation

  • (A): Points along the line indicate normality.
  • (B): A curved pattern is ideal.
  • (C): Random scatter indicates normality.
  • (D): None of the above.

Independence of Errors

  • Definition:
    Each error term should be independent; no autocorrelation should exist.
  • Checking Method:
    Plot residuals in order of data collection or use the autocorrelation function (ACF).

Visualizing Independence: Autocorrelation Function

acf(resid(model_example), main = "ACF of Residuals")
  • Interpretation:
    Significant auto-correlation suggests dependency among errors.

Quiz: Independence of Errors

  • (A): Independence means no auto-correlation.
  • (B): Independence means errors follow a normal distribution.
  • (C): Independence means errors have constant variance.
  • (D): Independence is not important in regression.

  • Purpose: Create Q-Q plots to assess normality.

  • Purpose:

Using boxcox() for Transformations

  • Purpose:
    Identify an optimal power transformation parameter \(\lambda\) to stabilize variance.
  • Mathematical Formula:
    \[ Y'(\lambda) = \begin{cases} \frac{Y^\lambda - 1}{\lambda} & \lambda \neq 0 \\ \ln(Y) & \lambda = 0 \end{cases} \]
library(MASS)
boxcox_result <- MASS::boxcox(model_lm, lambda = seq(-2, 2, 0.1))

Mathematical Foundations

Derivation: Residuals

  • Formula Recap:
    \[ e_i = Y_i - \hat{Y}_i \]
  • Mean Squared Error (MSE): \[ \text{MSE} = \frac{\sum_{i=1}^{n} e_i^2}{n-p} \]
  • Discussion:
    These formulas form the basis of diagnostic statistics in regression.

Derivation: Standardized Residuals

  • Formula:
    \[ z_i = \frac{e_i}{\sqrt{\text{MSE}}} \]

  • Purpose:
    Standardizes residuals for easier comparison and outlier detection.

Derivation: Box-Cox Transformation

  • Transformation: \[ Y'(\lambda) = \begin{cases} \frac{Y^\lambda - 1}{\lambda} & \lambda \neq 0 \\ \ln(Y) & \lambda = 0 \end{cases} \]

  • Objective:
    Stabilize variance and achieve normality by selecting \(\lambda\) that maximizes the log-likelihood.

Case-Deleted and Studentized Residuals

Introduction to Case-Deleted Residuals

  • Definition:
    Case-deleted residuals (also known as externally studentized residuals) are computed by fitting the model without a particular observation and then comparing the observed value to the prediction from that reduced model.

  • Mathematical Expression:
    The externally studentized residual is given by: \[ t_i = \frac{e_i}{s_{(i)}\sqrt{1-h_i}} \]

where:

  • \(e_i\) is the raw residual.
  • \(s_{(i)}\) is the standard error computed with observation \(i\) omitted.
  • \(h_i\) is the leverage of the \(i\)th observation.
  • Purpose:
    This measure provides a more accurate indicator of an observation’s influence by removing its own contribution to the variance estimate.

R Functions for Case-Deleted Residuals

  • Base R:
    • Use the function rstudent() to compute externally studentized residuals.
# Compute externally studentized residuals (case-deleted)
case_deleted_res <- rstudent(model_example)
head(case_deleted_res)
         1          2          3          4          5          6 
-1.3428426 -0.2312438 -1.1189360 -0.8180318 -0.6841502 -2.1703575
  • Interpretation:
    Values with high absolute magnitude (commonly above 2 or 3) indicate potentially influential observations.

Standardized vs. Studentized Residuals

  • Standardized Residuals:
    Computed as: \[ r_i = \frac{e_i}{s \sqrt{1-h_i}} \]

where \(s\) is the overall standard error estimate.

  • Studentized (Externally) Residuals:
    Computed as: \[ t_i = \frac{e_i}{s_{(i)}\sqrt{1-h_i}} \]

where \(s_{(i)}\) is the error standard deviation calculated by leaving the \(i\)th observation out.

  • Key Difference:
    Standardized residuals use the model’s overall error variance while studentized residuals account for the influence of each observation by recalculating the variance with that observation deleted.

R Code: Comparing Standardized and Studentized Residuals

# Compute standardized and studentized residuals using base R
std_resid_base <- rstandard(model_example)
stud_resid_base <- rstudent(model_example)
head(std_resid_base)
